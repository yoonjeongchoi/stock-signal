지난 10일간 Streamlit 플랫폼을 통해 'AI 주식 시그널 아카이브'의 기틀을 성공적으로 닦았습니다. 하지만 우리는 여기서 멈추지 않습니다. 더 많은 투자자에게 다가가고, 서비스로서의 완성도를 높이기 위해 오늘부터 **Next.js**로의 거대한 마이그레이션 대장정을 시작합니다. 오늘은 왜 우리가 플랫폼을 전환하며, 이를 통해 달성하고자 하는 비전이 무엇인지 기록해 봅니다.

## 왜 Next.js로 전환하는가?

Streamlit은 파이썬으로 빠르게 프로토타입을 만들기에 최적의 도구였습니다. 하지만 본격적인 웹 서비스를 지향하는 단계에 이르면 다음과 같은 세 가지 근본적인 한계에 직면하게 되었습니다.

첫째는 **SEO(검색 엔진 최적화)**의 부재입니다. Streamlit은 클라이언트 사이드 렌더링(CSR) 방식으로 작동하며, 검색 엔진 로봇이 페이지의 내용을 수집하는 데 제약이 큽니다. 시그널 아카이브에 쌓인 양질의 데이터들이 검색 결과에 노출되어 가치를 발휘하기 위해서는 서버 사이드 렌더링(SSR)이 필수적이었습니다.

둘째는 **광고 수익화와 비즈니스 모델**의 확장성입니다. 우리는 구글 애드센스(Google AdSense)나 에조익(Ezoic) 같은 광고 플랫폼을 연동하여 서비스의 지속 가능성을 확보하고자 합니다. Streamlit은 이러한 외부 광고 스크립트를 삽입하고 관리하는 기능이 제한적이기 때문에, 유연한 프론트엔드 프레임워크인 Next.js로의 전환이 불가피했습니다.

셋째는 **사용자 경험(UX)의 최적화**입니다. 모바일 환경에서의 완벽한 반응형 레이아웃, 세밀한 애니메이션 효과, 그리고 대용량 데이터를 처리할 때의 즉각적인 응답성 등을 구현하기 위해 현대적인 웹 기술 스택이 필요해진 시점입니다.

## Next.js 마이그레이션 계획: 기술적 로드맵

이번 마이그레이션은 단순히 껍데기만 바꾸는 작업이 아닙니다. 기존에 검증된 강력한 파이썬 백엔드 엔진은 유지하되, 사용자 접점인 인터페이스를 완전히 재정의하는 과정입니다.

1. **하이브리드 아키텍처 구축**: 파이썬 기반의 데이터 수집기(GitHub Actions 기반)는 그대로 운영하고, Next.js는 이를 API 형태로 호출하거나 JSON 파일에서 직접 데이터를 읽어 정적 페이지를 생성(SSG/ISR)하도록 설계했습니다.
2. **Vercel 플랫폼 활용**: 글로벌 배포의 최적화와 안정성을 위해 Vercel을 메인 호스팅 플랫폼으로 선택했습니다. 이를 통해 전 세계 어디서든 빠른 로딩 속도를 보장할 수 있게 되었습니다.
3. **Tailwind CSS를 통한 미려한 디자인**: 디자인 시스템을 새롭게 정립하여, '토스 증권' 스타일의 깔끔하고 직관적인 모바일 우선 UI를 구축할 예정입니다. 단순히 정보를 보여주는 것을 넘어, 보는 즐거움이 있는 서비스를 목표로 합니다.
4. **그로스 엔진 탑재**: 상세 페이지마다 메타 태그를 동적으로 생성하고, 사이트맵(Sitemap)과 로봇 파일(`robots.txt`)을 최적화하여 검색 유입을 극대화하겠습니다.

## 새로운 도전을 향해

오늘의 기록은 'AI 주식 시그널 아카이브'가 단순한 프로토타입 프로젝트를 넘어 진정한 서비스로 진화하는 첫날의 기록입니다. 플랫폼을 바꾼다는 것은 많은 시행착오와 기술적 난관이 따르겠지만, 그 과정에서 얻을 기술적 성장과 사용자들의 피드백이 벌써 기대됩니다.

우리는 이제 파이썬의 강력한 분석 능력과 Next.js의 유연한 서비스 능력을 결합한 최상의 결과물을 향해 달려갑니다. 기세는 여전히 최고조입니다. 다음 연재부터는 본격적인 Next.js 구현 과정과 변화된 UI의 모습을 하나씩 공개하겠습니다.
